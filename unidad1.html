1.2 Organización de los datos en la memoria física y lógica

La memoria RAM está físicamente formada por bits, componentes que pueden estar en uno de dos estados claramente definidos: encendido o apagado (ceros y unos). Como se observa, un bit puede representar dos valores; combinando grupos de bits se pueden representar más estados.

La memoria física RAM está compuesta por casillas en las cuales se pueden almacenar datos organizados por direcciones de memoria. Cada celda en una dirección puede ser de 8, 16, 32 o 64 bits.

1.2.1 Clasificación de datos simples y compuestos

De acuerdo con la forma en que los datos se organizan, se clasifican en:

  • a) Tipos de datos simples (primitivos).
  • b) Tipos de datos compuestos o estructurados.
a) Tipos de datos simples

Un tipo de dato simple hace referencia a un único valor a la vez y ocupa solo una casilla de memoria. En este grupo se encuentran:

  • Enteros (int): números sin decimales, como 10, -3 o 2026.
  • Flotantes (float o double): números con punto decimal, como 3.14 o -0.25.
  • Caracter (char): un solo símbolo, como 'a', '9' o '$'.
  • Booleanos (bool): representan valores lógicos: verdadero (true) o falso (false).
Direccionamiento de la memoria

La unidad de control se dirige a los distintos bytes de memoria por su posición. Todos los lenguajes de programación permiten dar un nombre a las posiciones de memoria, conocidas como variables. Una variable puede cambiar su contenido o almacenar un valor constante.

En C++, declaramos variables y reservamos memoria de la siguiente manera:

int Base = 3;
int Altura = 2; 
int Area;

Automáticamente, la unidad de control asigna a las casillas de memoria las direcciones correspondientes a esas variables. Al declararlas como tipo int, solo permitirán datos enteros. En este ejemplo, se les asignó un valor inicial a Base=3 y Altura=2.

b) Tipos de datos compuestos

Las estructuras de datos se caracterizan por usar un único nombre para hacer referencia a un grupo de casillas en memoria. Tienen varios componentes de tipos simples. Un ejemplo clásico son los Arreglos (arrays): colecciones ordenadas de elementos del mismo tipo.

1.3 Representación de las estructuras de datos

Las estructuras de datos son fundamentales para el desarrollo de grandes sistemas de software. Permiten organizar, almacenar y manipular datos de manera eficiente en la memoria de una computadora.

1.3.1 Operaciones básicas sobre las estructuras de datos

Las operaciones más comunes aplicables a la mayoría de las estructuras son:

  • + Insertar: Agregar un valor a la estructura.
  • - Eliminar: Borrar un valor existente.
  • ⌕ Buscar: Encontrar un valor específico.
  • ↻ Actualizar: Modificar un valor existente.
  • → Recorrer: Acceder a cada elemento secuencialmente.
